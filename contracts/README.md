## Deploy

```bash
# Uses Foundry encrypted keystore — no private key in .env
forge script script/Deploy.s.sol \
    --rpc-url base_sepolia \
    --account <keystore-name> \
    --broadcast \
    --verify \
    -vvvv
```

Output is written to `./deployments/base_sepolia.json`.

To redeploy only the verifier + attestation cache after a circuit update:

```bash
DKIM_REGISTRY=0xd984F26057A990a4f4de5A36faF7968b818BAe46 \
forge script script/RedeployVerifier.s.sol \
    --rpc-url base_sepolia \
    --private-key $DEPLOYER_PRIVATE_KEY \
    --broadcast -vvvv
```

## Contract Architecture

### `AttestationCache.sol`

The core registry. Enforces:

1. Valid Groth16 proof against the on-chain verification key
2. `pubSignals[2] == uint256(uint160(msg.sender))` — proof bound to caller, prevents front-running
3. DKIM pubkey hash registered in `DKIMRegistry`
4. Wallet has no existing attestation — one registration per wallet (`WalletAlreadyAttested`)

Storage is struct-packed for gas efficiency:

```solidity
struct Attestation {
    uint256 pubkeyHash;     // slot 1
    uint64  emailTimestamp; // slot 2 — packed with registeredAt
    uint64  registeredAt;   // slot 2
}
```

Inherits `Ownable2Step` + `Pausable`. The owner can pause `attest()` in case a circuit vulnerability is discovered. Ownership transfers require the new owner to call `acceptOwnership()`.

> **v2 note:** `nullifier` (Poseidon of the `To:` address) was removed in the partial-SHA circuit redesign. The `To:` header lives in the precomputed prefix and cannot be extracted inside the circuit. One-wallet-one-attestation is now enforced solely by the `WalletAlreadyAttested` check.

### `DKIMRegistry.sol`

Owner-controlled allowlist of Poseidon pubkey hashes for known DKIM signing keys.

```solidity
function setKey(uint256 pubkeyHash, bool valid) external onlyOwner;
function setKeys(uint256[] calldata hashes, bool valid) external onlyOwner;
function isValid(uint256 pubkeyHash) external view returns (bool);
```

Inherits `Ownable2Step`. Registered on Base Sepolia via `pnpm seed-dkim` — **52 keys** across 22 exchanges (last updated 2026-03-21):

**Live exchanges (21 keys):**
Coinbase (×2), Binance (×4), Kraken (×2), OKX (×2), Bybit, Gemini, Robinhood, Crypto.com, KuCoin (×6 including EngageLab)

**Defunct exchanges (31 keys — Rug Survivor Registry):**
FTX (×7), FTX US (×3), Voyager (×3), BlockFi (×2), Terra/Luna (×3), Anchor Protocol, Hodlnaut, WazirX (×6), Celsius, Vauld, Mt. Gox, QuadrigaCX (×2), DMM Bitcoin

#### How pubkeyHash is computed

The ZK circuit runs `PoseidonLarge(n=121, k=17)` over the 17 × 121-bit chunks of the RSA modulus. Because circomlibjs's Poseidon supports at most 16 inputs, the circuit merges consecutive chunk pairs first:

```
halfSize    = ceil(17 / 2) = 9
merged[i]   = chunk[2i] + 2^121 × chunk[2i+1]   for i = 0 … 7
merged[8]   = chunk[16]                           (unpaired last chunk)
pubkeyHash  = Poseidon(9)(merged)
```

Both 1024-bit and 2048-bit RSA keys are supported — the chunking always produces 17 values (upper chunks are zero for smaller keys).

#### Seeding the registry from DNS

`scripts/seed-dkim.ts` in the monorepo root queries DNS-over-HTTPS for every known exchange selector, computes the pubkeyHash using the same algorithm as the circuit, checks what is already registered on-chain, and batch-calls `setKeys()` with any new hashes.

```bash
# Preview — prints hashes, no on-chain writes
pnpm seed-dkim:dry

# Live run against Base Sepolia (default)
PRIVATE_KEY=0x<owner-key> pnpm seed-dkim

# Live run against Base Mainnet
PRIVATE_KEY=0x<owner-key> pnpm seed-dkim -- --network base
```

> **Amazon SES keys** — many exchanges deliver transactional email through Amazon SES with per-account, randomly-named selectors (e.g. `xmk7xv4dzq3ad3l3wrpegkiqkegm4nst._domainkey.info.coinbase.com`). These cannot be discovered via DNS without knowing the selector. Register them by extracting `pubSignals[0]` from a real proof and calling:
>
> ```bash
> cast send <DKIM_REGISTRY> "setKey(uint256,bool)" <pubSignals[0]> true \
>     --rpc-url https://sepolia.base.org \
>     --account <keystore-name>
> ```

#### Key rotation

DKIM keys do rotate, particularly for Google Workspace customers. Key rotation is handled automatically by the daily cron job (`.github/workflows/dkim-monitor.yml`) which runs `pnpm monitor-dkim` every day at 08:00 UTC. It detects new or rotated keys, registers them on-chain, and opens a GitHub issue for any DNS failures.

To run manually:

```bash
pnpm monitor-dkim:dry   # preview — no writes
PRIVATE_KEY=0x... pnpm monitor-dkim   # live
```

Old hashes are never removed — they remain valid for existing attestations signed before the rotation.

### `SignetGated.sol`

Abstract base contract for integrators. Inherit and call `requireSignet` or `requireSignetDomain`.

```solidity
import "@signet/contracts/SignetGated.sol";

contract MyAirdrop is SignetGated {
    uint256 public constant CUTOFF = 1704067200;

    function claim() external {
        IAttestationCache.Attestation memory a = requireSignetDomain(
            msg.sender, CUTOFF, COINBASE_PUBKEY_HASH
        );
        // a.emailTimestamp is available — no second cross-contract read needed
        emit Claimed(msg.sender, a.emailTimestamp);
    }
}
```

Both `requireSignet` and `requireSignetDomain` return the full `Attestation` struct, eliminating double cross-contract reads. Override `_signetAddress()` to point at a non-default deployment.

### `SignetAirdrop.sol` (example)

Production-ready reference implementation:

- `Ownable`, `Pausable`, `ReentrancyGuard`, `SafeERC20`
- All fields `immutable`
- Zero-address constructor guards
- `claim()` follows CEI pattern backed by `nonReentrant`
- `isEligible(address)` mirrors exact domain check from `claim()`
- `sweep()` for owner token recovery

### `SignetMerkleAirdrop.sol` (advanced example)

Two-layer sybil resistance: Merkle snapshot + Signet ZK attestation.

Leaf encoding is OZ-compatible: `keccak256(bytes.concat(keccak256(abi.encode(addr, amount))))`. Generate trees off-chain with `StandardMerkleTree.of(entries, ["address", "uint256"])`.

## Integration Quick Start

```solidity
import "@signet/contracts/SignetGated.sol";

contract AirdropClaim is SignetGated {
    uint256 public immutable CUTOFF;
    uint256 public constant  REQUIRED_DOMAIN = COINBASE_PUBKEY_HASH;
    address internal immutable _signet;

    mapping(address => bool) public claimed;

    constructor(uint256 cutoff, address signetCache) {
        CUTOFF  = cutoff;
        _signet = signetCache;
    }

    function _signetAddress() internal view override returns (address) {
        return _signet;
    }

    function claim() external {
        require(!claimed[msg.sender], "already claimed");
        requireSignetDomain(msg.sender, CUTOFF, REQUIRED_DOMAIN);
        claimed[msg.sender] = true;
        // distribute tokens
    }

    function isEligible(address wallet) external view returns (bool) {
        return !claimed[wallet] && isSignetEligibleDomain(wallet, CUTOFF, REQUIRED_DOMAIN);
    }
}
```

Or use the contract wizard at [signet.xyz/developers](https://signet.xyz/developers) to generate a pre-configured contract.
